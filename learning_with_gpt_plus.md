## GPT摸底测试  
### 数据结构是什么  
数据结构是数据元素之间关系的组织方式，以及对应操作方式  
让数据更高效地被操作而设计的组织形式  
### 数组和链表的区别  
数组——连续访问 随机访问快(随机存取，(按位置访问元素的时间复杂度总为O（1）)) 插删慢  
链表——非连续存储 随机访问慢 删查慢  
### 为什么链表插入快  
只改几个指针，数组需要挪动后面的元素  
（链表插入其实还真不一定快）  
比如我要在第1000个位置插入，我就需要先找到第999个节点  
正确的说法是已知插入位置时候，链表插入更快  
### 什么是时间复杂度  
时间复杂度不是实际的时间，而是增长趋势  
输入规模变大的时候 算法执行次数增长的趋势

```C
for(i=0;i<n;i++)
```

```C
for(i=0;i<n;i++)
```

## 困难的时间复杂度判断  
### 一直在对半砍的是log n  
```C
int i = n;
while(i > 1){
    i = i/2;
}
```
能对一个固定的数除几次，正好是对数的定义  
所以就是log n的时间复杂度  
所以也不一定是对半砍，只要是固定减少的百分比就可以  
后续查找中学习的对半查找就是如此  
### 平方根复杂度  
```C
for(int i = i; i*i <= n;i++)
```
关键就是循环停止条件  
```C
i*i <= n
```
所以最大就是i=√n  
### 指数复杂度  
每次调用像细胞分裂，自我调用两次直到n耗尽  
```C
void f(int n){
    if(n==0) return;
    f(n-1);
    f(n-1);
}
```
### 递归的log n  
```C
void f(int n){
    if(n <= 1) return;
    f(n/2)
}
```
依旧每次规模减半  
### nlog n  
经典例子是归并排序  
每层处理n个数据，但外层对半拆分  
### 常数、低阶项忽略原则  
只考虑数量级最大的即可  


# 线性表  
## 课前提问  
### 什么叫线性结构  
核心是除了第一个和最后一个元素外，每个元素有且仅有一个直接前驱和一个直接后驱  
关键词：有且仅有一个  
## 线性表  
站和队列的母体  
### 什么是线性表  
正式定义：具有相同数据类型的n个数据元素的有限序列  
相同数据类型  
混搭着来，一般不叫标准线性表  
n个数据元素  
n=0时 也是线性表 即空表  
有限  
因为计算机内存有限  
线性表不是无限长  
序列  
意味着有顺序  
### 逻辑结构和存储结构  
逻辑结构代表的是元素之间理论上的关系  
不管实际怎么存  
存储结构是元素在计算机里到底是怎么放的  
有顺序存储（数组）（地址连续）和链式存储（链表）（地址不连续，靠指针链接）  
### 顺序表实现  
比如数组  
```C
int a[5] = {10,20,30,40,50};
```
内存就是   
```
地址      内容
1000      10
1004      20
1008      30
1012      40
1016      50
```
一格一格挨着  
顺序表的本质就是用一组地址连续的存储单元一次存放线性表元素  
灵魂是地址连续  


为什么顺序表查找快  
因为计算机不用一个一个查找  
计算机自己直接用公式计算出元素地址  
元素地址 = 起始地址+元素大小*下标  
直接定位 随机存取 时间复杂度O(1)  
随机存取像坐电梯 顺序存取像爬楼梯  


为什么顺序表插入慢  
需要把插入元素之后的元素全部向后挪动  
删除同理  

### 考试高频 平均移动元素怎么算  
假设有n个元素  
```
A B C D E
```
n = 5  
可以插的位置是  
```
_A_B_C_D_E_
```
共六个位置可供插入  
（5+4+3+2+1+0）/6 = 2.5  
推广为n个元素时候  
总和就是  
```
n+(n-1)+(n-2)+...+0
```
公式  
```
n(n+1)/2
```
再除以位置数  
得到
```
n/2
```
所以插入的平均移动次数为n/2次  


再求删除平均移动次数  
还是
```
A B C D E
```
五个元素 移动次数为4+3+2+1+0  
一共有五个数字可以删，也就是五种情况  
推广到n个元素得到  
```
(n-1)/2
```
### 链表  
为什么需要链表  
我的回答：插入和删除时都消耗大量时间  
真正本质是  
顺序表要求连续内存  
如果在数组一开始找到的那块连续的内存空间用光了，但是还想继续插入新元素  
就需要  
1.找一块更大的连续内存  
2.把全部元素复制过去  
3，再插入  
所以，链表真正的核心思想就是摒弃对于连续内存的要求  
直接不要求连续，依靠指针维系逻辑关系  
链表：逻辑连续，物理不连续  
节点（Node）  
链表不直接存元素，而是存节点  
节点 = 数据 + 指针  
例如
```C
typedef struct Node{
    Elemtype data;
    struct Node *next;
}Node;
```
data存数据  
next存下一个节点地址  
所以链表插入快  
只需要更改指针  
把
```C
A->next = C;
```
改成
```C
A->next = B;
B->next = C;
```
不用移动数据  
但是注意前提  
比如：  
插到第1000个位置。  
先找：  
第999个节点：  
```
O(n)
```
再插：  
```
O(1)
```
总：  
```
O(n)
```
所以精确表述：  
已知插入位置时，链表插入 O(1)  


费曼问题：为什么链表不能随机存取  
因为链表随机找出的元素的地址无法像顺序表一样直接计算出来，必须要从头节点开始通过指针一个一个向后遍历式寻找  
## 手搓代码环节  
### 定义一个单链表节点  
自己写的  
```C
#include<stdio.h>
#include<stdlib.h>

typedef int Elemtype

typedef struct{
    Elemtype data;
    struct Node* next;
}Node;
```
GPT提出的问题  
1.typedef后面一定要加分号  
2.习惯是用ElemType作为数据类型  
3.应当是
```C
typedef struct Node{
    Elemtype data;
    struct Node* next;
}Node;
```
### 费曼问题一
typedef struct后面一定要加Node  
**原因1**：C编译器是从上往下逐行看的  
如果不加Node  
编译器会先读到一个匿名结构体  
然后读到  
```C
struct Node *next;
```
的时候直接疑惑，struct Node是什么  
因为我在上文中并没有定义过  
这叫做**声明顺序**问题  
**原因2**  
Node和struct Node根本不是一个东西  
比如：
```C
struct Student{
    int age;
};
```
之后我只能写  
```C
struct Student s;
```
不能直接
```C
Student s;
```
因为没有定义过Student
只有
```C
typedef struct Student{
    int age;
}Student;
```
也就是通过typedef将整个结构体（包含struct）整体重新命名后才成为别名  
所以
```C
struct Node
```
是结构体的tag（标签名）
```C
Node
```
是typedef出来的别名  
二者不一样  
在定义好了结构体之后的程序中，可以直接写Node *p;  


我的写法：
```C
typedef struct{
```
相当于：  
我造了一个“没名字的人”  
然后我说：  
struct Node *next;  
相当于：  
去找一个叫 Node 的人  
编译器：  
哪有这个人？  


标准写法： 
```C 
typedef struct Node{
```
相当于:  
我现在创造一个叫 Node 的人  
下面：  
```C
struct Node *next;
```
编译器：  
好，我认识。  
### 费曼问题二  
为什么
```C
Node next;
```
会出问题，而   
```C
Node *next;
```
不出问题  
我的回答：Node next;会导致Node本身的定义无限套娃   
使得编译器无法分配内存  
而Node* next作为结构体指针的内存大小是固定的  

提问：为什么这里的Node next也好Node *next也好都不加struct了，按照顺序编译的原则，此时typedef出来的别名Node还没出现啊？ 
GPT的回答：是他马虎写错了   
### 链表初始化  
**带头节点初始化**  
我自己写的
```C
Node* InitLink(){
    Node *head = (Node*)malloc(sizeof(Node));
    head->next = NULL;
}
```
GPT点评：基本正确唯一错误的是忘记return了  
还有就是如果追求严谨，需要增加malloc失败的判定
```C
Node *InitLink(){
    Node *head = (Node*)malloc(sizeof(Node));
    if(head == NULL){
        return NULL;
    }
    head->next = NULL;
    return head;
}
```
**不带头节点初始化**  
```C
Node *InitLink(ElemType e){
    Node *head = (Node*)malloc(sizeof(Node));
    if(head == NULL){
        return NULL;
    }
    head->data = e;
    head->next = NULL;
    return head;
}
```
