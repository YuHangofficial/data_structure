## GPT摸底测试  
### 数据结构是什么  
数据结构是数据元素之间关系的组织方式，以及对应操作方式  
让数据更高效地被操作而设计的组织形式  
### 数组和链表的区别  
数组——连续访问 随机访问快(随机存取，(按位置访问元素的时间复杂度总为O（1）)) 插删慢  
链表——非连续存储 随机访问慢 删查慢  
### 为什么链表插入快  
只改几个指针，数组需要挪动后面的元素  
（链表插入其实还真不一定快）  
比如我要在第1000个位置插入，我就需要先找到第999个节点  
正确的说法是已知插入位置时候，链表插入更快  
### 什么是时间复杂度  
时间复杂度不是实际的时间，而是增长趋势  
输入规模变大的时候 算法执行次数增长的趋势

### 问题二  
为什么：  
```C
for(i=0;i<n;i++)
```
是 O(n)  
而：
```C
for(i=0;i<n;i++)
    for(j=0;j<n;j++)
```
是 O(n²)  
## 困难的时间复杂度判断  
### 一直在对半砍的是log n  
```C
int i = n;
while(i > 1){
    i = i/2;
}
```
能对一个固定的数除几次，正好是对数的定义  
所以就是log n的时间复杂度  
所以也不一定是对半砍，只要是固定减少的百分比就可以  
后续查找中学习的对半查找就是如此  
### 平方根复杂度  
```C
for(int i = i; i*i <= n;i++)
```
关键就是循环停止条件  
```C
i*i <= n
```
所以最大就是i=√n  
### 指数复杂度  
每次调用像细胞分裂，自我调用两次直到n耗尽  
```C
void f(int n){
    if(n==0) return;
    f(n-1);
    f(n-1);
}
```
### 递归的log n  
```C
void f(int n){
    if(n <= 1) return;
    f(n/2)
}
```
依旧每次规模减半  
### nlog n  
经典例子是归并排序  
每层处理n个数据，但外层对半拆分  
### 常数、低阶项忽略原则  
只考虑数量级最大的即可
